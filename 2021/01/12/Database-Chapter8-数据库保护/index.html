Database-Chapter8-数据库保护

Posted on 2021-01-12 In Database , Web Valine:

全文共732字，推荐阅读时间7~10分钟。

安全性

定义

  • 安全性(security)：防止不合法的使用造成的数据泄露、更改、破坏。

控制

  • 用户标识和认证（最外层）

    • 常用：用户名和密码（口令）

  • 存取控制

    • 权限定义

    • 权限检查

    • 方法

      • 自主(DAC)：用户可以将权限转授

        • 权限=数据对象+操作类型

        • SQL

          • 用户级：操作数据库，例如创建外模式、创建会话、创建表等。
          • 关系级：操作某个关系和视图，针对具体的对象，例如在某个表中进行CRUD操作。

        • 角色：一组权限的集合，可以授给用户或其他角色。

    • 强制(MAC)：不同数据对象有不同的加密等价，每个用户的许可证级别决定了能够访问哪些数据，由系统直接指定级别。

      • 实体=主体+客体
      • 主体：实际用户
      • 客体：系统中的被动实体，包括文件、基本表、索引、视图等。
      • 许可证级别(Label，敏感度标记)
        • 主体大于等于客体：读
        • 主体等于客体：写

  • 操作系统/网络安全保护

  • 数据加密

  • 其他

    • 视图：不同用户有不同的视图
    • 审计：将用户对数据库的操作记录在审计日志中，出现安全问题时进行审查。

可信系统评测

  • TCSEC
  • TDI

完整性

定义

  • 完整性=正确性+相容性
    • 正确性：类型、取值范围
    • 相容性：同一个事实的值应该相同
  • 确保数据库语义正确

约束条件

  • 作用对象
    • 列：对属性进行约束
    • 元组：元组间的字段约束（属性间约束）
    • 关系：元组之间、关系之间的约束，例如实体完整性、参照完整性。
  • 时机
    • 静态：数据库在每一确定状态应该满足的约束
    • 动态：新、旧值之间的约束，例如年龄只能上升不能下降。

总共可以组合出3*2=6种约束条件

控制

  • 定义

  • 检查

    • 立即执行：语句执行完后立即检查
    • 延迟执行：事务执行完毕后再进行检查

  • 违约相应

  • 规则：$(D,O,A,C,P)$

    • D：数据本身
    • O：操作
    • A：断言

    • C：A指定的谓词（自然语言约束的主语）

    • P：违反规则时触发的过程

  • SQL

    • CREATE TABLE

    • CREATE ASSERTION

    • CREATE TRIGGER

Reference

  • BUAA Database Theory Lecture, Autumn of 2020.
  • Shan Wang, Shixuan Sa, Database System Syllabus(5th Edition), Advanced Education Express.
