全文共2102字，推荐阅读时间15~20分钟。

数据管理技术发展

  • 人工管理

    • 数据不在计算机上保存

    • 程序决定数据的逻辑结构和物理结构，数据与程序之间不具有独立性。

      要改变数据的结构就要改变程序

    • 数据面向应用，程序之间不能共享数据。

  • 文件系统

    • 存取方法作为中介出现

    • 数据以文件形式保存在外存上

    • 数据的存取以记录为单位

    • 程序和数据具有一定的独立性

    • 缺点：

      • 数据冗余度大（因为数据不能被程序共享，即不能够同时进行操作，必须创建多个副本），可能带来数据的不一致性。
      • 数据和程序之间的独立性依然不明显，缺乏逻辑独立性。（改文件逻辑结构时还要修改程序）

      文件系统是不具弹性的无结构的数据集合，数据之间（不是程序和数据）是孤立的，不能反映现实世界事物之间的内在联系。

      • 不具弹性：改变数据的结构就得改程序
      • 无结构：数据之间孤立

  • 数据库系统

    • 数据结构化：描述数据时，不仅描述数据本身，还要描述数据之间的联系。

      数据结构化是数据库与文件系统的根本区别

    • 数据冗余度小，易扩充。

    • 数据和程序的独立性较高

      • 数据独立性

        • 物理：数据的物理结构改变时，数据的逻辑结构可以不变，从而程序也不用改变。
        • 逻辑：数据的逻辑结构改变时，程序不用改变。

    • 数据库映象：不同结构之间数据的转换关系

      物理结构发生变化时，可以通过调整数据库映象来确保逻辑结构不变。

    • 统一的数据控制功能

      • 安全性控制

      • 完整性控制：数据的正确性和相容性

        相容性：同一条数据在不同的程序应该是相同的

      • 并发控制

      • 恢复

      • 存取单位是数据项

        可以存取一组记录，也可以取一组数据项。

        数据项的粒度小于文件

        Q：数据项和记录的区别

        A：数据项是列，记录是行。

数据模型

  • 用于抽象和表示现实世界中的数据和信息
  • 层次

    • 数据模型（层次、网状、关系模型）

层次

  • 信息世界就是概念模型：E-R图
  • 机器世界就是数据模型：数据表

概念模型

  • 基于信息世界的主要概念，表达语义。
  • 语义表达能力强
  • 清晰
  • 最常用的表示方法：E-R法(Entity-Relation)

基本概念

  • 实体(Entity)：客观存在并可以相互区分的食物，在E-R图中为矩形。

  • 属性(Attribute)：实体具有的某一特性，在E-R图中为椭圆形。

  • 码(Key)：唯一的标识实体的属性集（至少一个，最多由属性集大小确定）

  • 域(Domain)：某些属性的取值范围

  • 实体型(Entity Type)：表示一类实体(Class)

    • 用名称和属性集表示

  • 联系(Relation)：实体型之间的联系，在E-R图中为菱形。

    • 名称：产品由若干零件构成
    • 类型
      • 一对一(1:1)
      • 一对多(1:n)
      • 多对多(m:n)
    • 联系可以具有属性，例如产品-组装-零件中，组装可以有零件数属性。
    • 联系是有方向的，一对多不等价于反向的多对一。

E-R

  • 用E-R图描述现实世界
  • 将E-R图转换为数据模型
组成
  • 实体
  • 属性
    • 作为Key的属性
      • 在属性下画线，多个属性就都画。
      • 在无向边上画两条杠
      • 组合属性作为Key：在两条无向边之间画一条弧之后，在弧上画两条杠。
  • 联系：无向边旁需要标注联系的类型（三选一）
    • 可能存在自相联关系
    • 三个或多个实体可能通过一个联系建立关系
    • 两个实体之间可能存在多种联系
      • 工人使用/维修仪器
语义
  • 存在依赖：有向边指向弱实体（依赖于其他实体的实体）
    • 弱实体是双层框
  • 标识依赖：有向边指向被标识实体（不能由自己的属性来唯一标识，需要和其他实体的属性组合起来作为Key）
  • 实体的子类：无向边上画一个圆
    • 子类是外侧框加两条竖线
    • 子类的交不一定为空

数据模型

  • 数据结构
  • 数据操作
  • 完整性约束

数据结构

  • 描述对象的相关性质
  • 描述对象之间的联系，如关系模型中的关系。
  • 数据静态特征的描述
  • 三要素中最重要的方面

数据操作

  • 操作及操作规则

  • 操作符号、操作规则、操作语言

    SQL就是一种操作语言

  • 数据动态特征的描述

  • 数据库中主要有检索和更新两大类操作

约束条件

  • 完整性规则的集合
  • 模型通用的完整性约束条件
  • 针对具体数据的特定语义的约束条件

分类

  • 层次模型：树
    • 最早出现
    • 有向树，连线表示一对多的联系（包含一对一）。
    • 结构简单，效率高，但是支持的联系的种类太少（只能直接表示二元一对多联系）
  • 网状模型：图
    • 有向图，连线表示一对多的联系（包含一对一）。
    • 结构复杂，效率高。
  • 关系模型：二维表
    • 用二维表格表示实体及实体之间的联系
    • 规范化关系：每个分量（域）是不可分割的数据项，并且不能出现表中套表的情况。
    • 有数学基础

数据库系统结构

  • 三级模式
    • 外模式
    • 模式
    • 内模式
  • 两级映象
    • 外模式/模式映象
    • 模式/内模式映象
  • 优点
    • 数据独立性
    • 简化用户接口
    • 有利于数据共享（外模式可以相互覆盖）
    • 利于数据的安全保密

模式

  • 逻辑模式/概念模式
  • 全体数据的逻辑结构和特性的描述，例如所有关系模式构成的集合。
  • 三级模式的核心
  • 模式描述语言(DDL, Data Description Language)

外模式

  • 个别用户的数据视图（与某一应用有关的数据逻辑表示）

  • 通常是模式的子集（子模式）

  • 不同的外模式可以相互覆盖，一个应用只能启用一个外模式。

    外模式和应用是一对多的关系

  • 外模式DDL，通常和应用具有相同的语法。

内模式

  • 存储模式，数据的物理结构和存储方式的描述。
  • 内模式DDL

两级映象

  • 数据逻辑独立性：外模式/模式映象
  • 数据物理独立性：模式/内模式映象

DBMS

主要功能

  • 数据库的定义功能
    • 使用DDL语言描述外模式、模式、内模式（源模式）
  • 数据存取功能
    • 提供DML(Data Manipulation Language)语言进行CRUD
  • 数据库运行管理
    • 并发控制
    • 存取控制
    • 完整性
    • 日志组织和管理
    • 事务管理
    • 自动恢复

组成

  • 语言编译处理程序
  • 系统运行控制程序
  • 系统建立和维护
  • 数据字典（数据目录）：由若干个表组成

  • 用库
