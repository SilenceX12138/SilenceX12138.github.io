Chapter6 关系查询处理与查询优化

全文共448字，推荐阅读时间3~5分钟。

概述

  • 分析
  • 检查
    • 将查询转换为查询树（语法分析树）
  • 优化
    • 代数：恒等变换
    • 物理：根据数据规模、分布等确定对应的底层存储策略。
  • 执行

优化

  • 查询代价=I/O代价（主要）+CPU代价+内存代价

代数优化

  • 变换规则：主要是优化连接（笛卡尔积为主）运算。
    • 交换律
    • 结合律
  • 特殊优化：先缩小范围再运算一般效率更高
    • 投影、选择的串接：合并操作减小遍历次数
    • 投影、选择的交换律：先投影再选择
    • 选择、笛卡尔积的交换律：能先投影时，先投影再作笛卡尔积。
  • 查询树的启发式优化
    • 减少中间（临时）关系：选择、投影尽可能先做。
    • 减少扫描次数：投影、选择并行等操作。
    • 减少操作总数：连接=笛卡尔积+选择
    • 中间结果复用：使用中间文件存储公共表达式的结果

物理优化

  • 存取路径选择

    • 选择操作

      • 小关系：全表扫描
      • 大关系：索引扫描

    • 连接操作

      • 排序-合并

      • 索引连接：为连接属性建立索引

      • Hash Join：为小表建Hash文件

      • 嵌套循环：小表在外层

        原因：分配时通常给外表的内存更大

        • 设外表分配$(K-1)$块，内表$1$块。

          $Cost=Cost_{out}+Cost_{in}*\frac{B_{out}}{K-1}$

步骤

  • 将查询转换成语法树
  • 代数优化
  • 物理优化，确定底层的存储路径
  • 生成查询计划

Reference

  • BUAA Database Theory Lecture, Autumn of 2020.
