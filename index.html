Ubuntu18.04: no wifi adapter found 解决方案

重要的事说在前面

很重要！很重要！很重要！

以下教程仅适用于intel系列无线网卡,至于其他例如broadcom等，只能提供一个参考思路。

Problem Description

terminal中输入lspci,得到类似如下结果

其中Network controller本不应该对应笔记本的无线网卡，但是ubuntu系统和intel系列网卡兼容性一言难尽，导致系统误把网卡作为了controller。

Solution

  1. 升级系统内核(optional)

    具体可以参见这篇博客，写的很好。

    虽然这一步是选择性的，但还是建议内核版本5.3以下的朋友升级到最新版本，少点bug不吃亏。

    还有，intel官网上写的驱动的kernel版本要求似乎有点问题，笔者5.0.0内核版本亲测：必须要升级到5.3+后才能识别“写着4.14+以上均适用”网卡驱动。

    总而言之，建议大家尽量还是选择升级内核版本。

  2. 安装网卡驱动

    • 首先是确认网卡型号

      建议在windows下按照这篇博客进行操作

    • 登陆intel驱动官网:https://wireless.wiki.kernel.org/en/users/drivers/iwlwifi![image-20200203110455667](/home/silence/.config/Typora/typora-user-images/image-20200203110455667.png)

    • 查找到对应的型号后下载.tgz文件

    • 解压文件

    • 安装驱动

    • 重启

    • 和wifi愉快拥抱吧～
