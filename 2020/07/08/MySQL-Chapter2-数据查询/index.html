MySQL-Chapter2-数据查询

Posted on 2020-07-08 In MySQL , Web Valine:

全文共1349字，推荐阅读时间7~10分钟。

基本查询

  • SELECT：表示要执行一个查询
  • *：表示所有列
  • FROM：表示将要从哪个表查询、
  • SELECT 1;常用来测试数据库连接
  • 查询结果是一个二维表

条件查询

  • WHERE引出条件

  • AND/OR/NOT进行逻辑运算

  • =（一个等号）表示相等判断

  • <>表示不等判断（等价于NOT xxx = xxx;）

  • ()进行条件分组

特殊操作

  • BETWEEN...AND...（闭区间）

  • LIKE判断相似

    • %表示任意字符，可以为空字符。

投影查询

只查询某些列的操作被称为投影查询

  • 为列增加别名

    SELECT row1, row2 name2, row3 FROM ...

排序

  • 数据库的查询结果默认按照id（主键）排序

  • 使用ORDER BY子句将记录根据字段值从低到高进行排序（默认升序ASC）

    增加DESC表示倒序

  • 多要素排序：将要素依次列出，逗号分隔即可。

  • 多行查询语句（关键词换行）

  • ORDER BY放在查询语句的最后

分页查询

当查询结果量很大时，进行分页显示。

  • 实质上就是从结果中截取一部分进行显示，进行多次查询即可得知所有结果。

    • LIMIT M OFFSET N：从N开始的最多M条记录（少于M条也不会报错）
    • 可以简写为LIMIT M, N
    • LIMIT M默认N为0

聚合查询

对于总数、平均数的计算，SQL内置了聚合函数，使用聚合函数进行查询就叫聚合查询。

  • 查询表中记录总数

    为返回的表设置别名

  • 查询特定记录总数

    • COUNT(*)和COUNT(id)等价
    • 没有匹配到行返回0

其他聚合函数

  • SUM：计算某一列的和

    • 没有匹配到行返回NULL

  • AVG：计算某一列的平均值

    • 没有匹配到行返回NULL

  • MAX：计算某一列的最大值

    • 没有匹配到行返回NULL

  • MIN：计算某一列的最小值

    • 没有匹配到行返回NULL

后两个不限于数值类型

  • FLOOR：向下取整函数

  • CEILING：向上取整函数

分组

  • 根据字段值自动分组计数（不用多次查询）

    • 为了区分输出的结果是哪个班的，可以多SELECT一个字段。

      • 注意只能SELECT用于分组的字段，加入其他字段可能会出现一个框内多个值的情况，MySQL会报错。

  • 多字段分组

  • 查询各个班级的平均分

  • 查出各个班级男生和女生的平均分

多表查询

查询两个表的乘积，又称笛卡尔查询。

  • 将两张表的行逐一进行枚举组合后拼在一起返回

  • 结果集列数是两张表列数之和

  • 结果集行数是两张表行数之积

  • 查询时为了区分列，可以设置别名。（注意字段名的访问方式）

    当表名过长时，可以选择给表再起一个别名。

连接查询

对多个表进行JOIN运算：先确定一个主表，然后从其他表中选择性地提取一些行加入到结果集中。

  • 查询学生信息的同时需要知道班级id对应的班级名

    • 内连接

      • 确认主表(students)
      • 确认连接表(classes)
      • 确认连接条件(ON)：当条件满足时把主表和连接表的对应行连接起来
      • 可以加上查询语句等辅助功能

    • 外连接

      相比于内连接会多出一行

      • RIGHT OUTER JOIN：返回右表（连接表）中存在的行（左表不存在无所谓）
      • LEFT OUTER JOIN：返回左表（主表）中存在的行（右表不存在无所谓）
      • FULL OUTER JOIN：两边表只要有一个存在就返回（RIGHT和LEFT的并集）
      • INNER JOIN：返回两张表都存在的记录
